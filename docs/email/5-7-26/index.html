Google / Gmail rejection

Fix 5.7.26

Gmail typically uses 5.7.26 when it can’t verify your sender as authenticated (or your policy requirements aren’t met). The fix is usually in SPF/DKIM/DMARC.

What usually causes this?
  • • No valid DMARC record published (or syntax invalid)
  • • SPF failing (missing includes, PermError, too many DNS lookups)
  • • DKIM missing, failing, or misaligned with the From: domain
  • • Using a new sending service without correct auth setup
  • • Forwarding / relaying scenarios breaking authentication

The fastest way to be sure is to scan the domain + the exact bounce text.
